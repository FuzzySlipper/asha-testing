# Demo Proof M3 Browser Interactive Gameplay Closeout

M3 establishes a browser-visible interactive demo proof path for `asha-demo`.
It starts from an independently launchable browser page, maps real DOM input
events to typed public ASHA request DTOs, records browser-local gameplay
readbacks, correlates those input/readback sequences with replay-style evidence
refs, and closes the chain with an aggregate proof gate.

## Commands

- `npm run browser:demo` writes the standalone launch target at `harness/out/browser-demo/latest/index.html` and `harness/out/browser-demo/latest/index.json`.
- `npm run browser:input-proof` loads the launch target in headless Chromium, dispatches DOM keyboard/pointer/wheel events, and writes `harness/out/browser-input-proof/latest/index.json`.
- `npm run browser:input-correlation` writes the browser input replay and evidence correlation artifacts under `harness/out/browser-input-replay/latest/` and `harness/out/browser-input-correlation/latest/`.
- `npm run browser:interactive-proof` is the M3 aggregate gate and writes `harness/out/browser-interactive-proof/latest/index.json`.

## Evidence Shape

The aggregate artifact records child artifact hashes, input event counts, typed
request counts, gameplay readback counts, replay hashes, and boundary-check
status. Sequence ids are required to line up across browser input events, typed
ASHA request DTOs, and browser-local gameplay readbacks.

## Boundaries

This is a browser interaction proof, not a runtime authority claim. M3 does not
claim native runtime authority, command-authority replay, hardware GPU evidence,
performance evidence, store submission, installer readiness, or Studio debug UI
correctness. The browser page uses public ASHA package-root evidence generated by
Node-side scripts and does not depend on `asha-studio` or arbitrary command
hatches.
