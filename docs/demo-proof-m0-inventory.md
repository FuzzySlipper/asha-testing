## Guardrails

- Use public ASHA package roots only: `@asha/contracts`,
  `@asha/game-workspace`, `@asha/runtime-bridge`, and approved Studio packages.
- Do not import raw native/WASM transports from consumers.
- Do not add arbitrary JSON command hatches.
- Do not let Studio become the source-of-truth asset database or runtime authority.
- Do not infer WASM, hardware GPU, performance, store submission, installer, or
  signing readiness from selected native backend proof.
- Generated `harness/out/**` artifacts are proof outputs; committed source files
  and public package APIs remain the durable contract inputs.
